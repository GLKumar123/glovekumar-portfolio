Introduction

Production incidents can occur at any time. A structured troubleshooting approach helps engineers identify root causes quickly and restore services with minimal downtime.

1. Troubleshooting Pods

  • Check Pod Status
  • Describe the Pod
  • Inspect Logs
  • Verify Events
  • Check Resource Limits
  • Verify Readiness & Liveness Probes
Terminal 
kubectl get pods -A

kubectl describe pod <pod-name>

kubectl logs <pod-name>

kubectl logs <pod-name> --previous

kubectl top pod
Common Issues
  • CrashLoopBackOff
  • OOMKilled
  • Pending Pods
  • ImagePullBackOff
  • Readiness Probe Failed

2. Troubleshooting Services

  • Verify Service exists
  • Check Service Endpoints
  • Verify Labels & Selectors
  • Confirm TargetPort mapping
Terminal 
kubectl get svc

kubectl describe svc my-service

kubectl get endpoints my-service

kubectl get pods --show-labels

3. DNS Troubleshooting

  • Verify CoreDNS
  • Test DNS Resolution
  • Inspect CoreDNS Logs
  • Check DNS Policies

kubectl exec -it busybox -- nslookup kubernetes.default

kubectl get pods -n kube-system

kubectl logs -n kube-system deployment/coredns

4. Ingress Troubleshooting

  • Validate Host Rules

  • Check Backend Service
  • Review Controller Logs

kubectl logs -n ingress-nginx deployment/ingress-nginx-controller

Common Production Problems

Component Typical Issue Action

Useful Kubernetes Debugging Commands

Production Kubernetes Cheat Sheet 
# Cluster
kubectl cluster-info
kubectl get nodes -o wide
kubectl top node

# Pods
kubectl get pods -A
kubectl describe pod 
kubectl logs 
kubectl logs  --previous
kubectl exec -it  -- sh
kubectl top pod

# Deployments
kubectl get deploy
kubectl rollout status deployment/
kubectl rollout history deployment/
kubectl rollout undo deployment/

# ReplicaSets
kubectl get rs

# Services
kubectl get svc
kubectl describe svc 
kubectl get endpoints 

# DNS
kubectl exec -it busybox -- nslookup kubernetes.default
kubectl exec -it busybox -- dig google.com
kubectl logs -n kube-system deployment/coredns

# Ingress
kubectl get ingress
kubectl describe ingress 

# Networking
kubectl get networkpolicy
kubectl get pods -n kube-system
kubectl get events --sort-by=.metadata.creationTimestamp

# Resources
kubectl top pod
kubectl top node

# Events
kubectl get events -A

# YAML
kubectl get pod  -o yaml
kubectl get svc  -o yaml
kubectl get ingress  -o yaml

Best Practices

  • Monitor applications with Prometheus and Grafana.
  • Configure Liveness and Readiness probes correctly.
  • Use resource requests and limits.
  • Centralize logs using OpenSearch, ELK, or Loki.
  • Create alerting rules for Pod restarts and node health.
  • Maintain production runbooks.
  • Practice troubleshooting in staging before production.
